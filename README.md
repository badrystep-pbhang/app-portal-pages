# app-portal-pages

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面可作为独立入口、工具界面或信息展示页使用。仓库本身不针对任何特定域名或网站，仅作为静态 HTML 资源的集合与版本管理。

## 目录说明

```
app-portal-pages/
├── pages/           # 存放所有 HTML 页面文件
├── assets/          # 公共资源文件（如 CSS、JS、图片等）
├── index.html       # 仓库首页（可选，用于展示页面列表）
└── README.md        # 本文件
```

- `pages/`：每个 HTML 页面建议以独立子目录或文件形式存放，便于维护。
- `assets/`：存放页面间共享的资源，避免重复引用。
- 可根据需要自行扩展目录结构，保持整洁即可。

## 页面归档说明

- 每个页面应独立、自包含，尽量减少对外部资源的强依赖。
- 建议在页面头部添加注释，说明页面用途、创建日期及维护人。
- 页面命名应简洁、有语义，避免含混或过长。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件即可预览。
3. 如需部署，可将整个仓库内容上传至任意静态托管服务（例如 GitHub Pages 或其他静态服务器）。

## 维护说明

- 本仓库欢迎提交新的独立 HTML 页面，但请确保页面内容合规、不含恶意代码。
- 提交前请确保页面在主流浏览器中可正常显示。
- 不建议在页面中嵌入跟踪脚本、广告或其他第三方主动加载内容。
- 如需修改现有页面，请遵循原有风格与结构。
